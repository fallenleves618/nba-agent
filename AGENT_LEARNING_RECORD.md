# Agent Learning Record

更新时间：2026-04-22

## 目的

这个文件用来长期记录本项目的 Agent 学习与改造路线。

目标不是单纯继续堆功能，而是通过持续改造 `nba-agent`，系统掌握以下能力：

1. Workflow：理解 Agent 是“流程 + 状态 + 工具 + 模型”的组合。
2. Prompting：理解 Prompt 是任务规格，而不是随意试词。
3. Tool Use：理解模型如何结合外部数据源工作。
4. Memory：理解 Agent 如何根据用户偏好做个性化。
5. Evaluation：理解如何客观比较 Prompt、模型和策略。
6. Reliability：理解真实 Agent 系统如何处理失败、超时和噪声。

## 学习总路线

### 第一阶段：看懂 Agent

学习目标：

- 理解当前 Agent 的工作流
- 理解每个模块职责
- 理解为什么慢、为什么失败、为什么保留/丢弃

当前阶段状态：`已完成`

已完成项：

- [x] 运行进度输出
- [x] 运行耗时展示
- [x] 总结输入 Top 展示
- [x] Agent 评分 / 理由展示
- [x] 基础失败诊断

本阶段结论：

- Agent 不是单个模型
- Agent 是采集、规则、模型、报告、推送组成的工作流

### 第二阶段：学会调 Agent

学习目标：

- 学会管理 Prompt
- 学会构建评估集
- 学会用指标比较不同 Prompt / 模型 / 参数

当前阶段状态：`进行中`

已完成项：

- [x] Prompt 版本管理
- [x] `filter_v1 / filter_v2`
- [x] `summary_v1 / summary_v2`
- [x] 本地评估集
- [x] `python3 -m nba_agent.app --eval`

待完成项：

- [x] 扩充评估集到 30-50 条
- [x] 支持一次对比多个 Prompt 版本
- [x] 支持输出更细的筛选误判分类
- [ ] 支持评估结果落盘保存

本阶段结论：

- 没有 eval，就无法稳定优化 Agent
- Prompt 应当版本化管理，而不是硬编码在代码里

### 第三阶段：学 Tool Use

学习目标：

- 理解事实源和社区源的区别
- 理解模型为什么要结合工具而不是只总结文本

当前阶段状态：`未开始`

计划项：

- [x] 将“事实源摘要”和“社区热议摘要”拆成两个区块
- [x] 提高官方源权重
- [x] 让总结显式引用比分、赛程、伤病等事实源
- [x] 为不同来源增加可信度标签

预期学习收获：

- Tool Use 的价值不在于“调用很多接口”
- Tool Use 的核心是“用外部事实约束模型输出”

### 第四阶段：学 Memory

学习目标：

- 理解 Agent 如何根据偏好进行个性化输出

当前阶段状态：`未开始`

计划项：

- [ ] 增加用户偏好配置文件
- [ ] 支持关注球队/球员/主题
- [ ] 支持排序权重按偏好调整
- [ ] 支持排除不想看的内容类型

预期学习收获：

- Memory 不一定复杂
- 很多 Agent 的记忆本质上是用户配置与历史状态

### 第五阶段：学 Multi-Agent / Multi-role

学习目标：

- 理解为什么要按角色拆分 Agent

当前阶段状态：`未开始`

计划项：

- [ ] 拆分 Scout：负责找候选内容
- [ ] 拆分 Judge：负责判断是否保留
- [ ] 拆分 Editor：负责总结
- [ ] 拆分 Fact-check：负责可信度标记

预期学习收获：

- Multi-Agent 不等于更复杂
- 角色拆分的核心价值是清晰、可替换、可调试

### 第六阶段：学 Reliability

学习目标：

- 理解真实 Agent 系统如何长期稳定运行

当前阶段状态：`未开始`

计划项：

- [ ] 模型失败时更细粒度回退
- [ ] 推送失败重试或缓存
- [ ] 成本与调用次数统计
- [ ] 每日失败率与时延统计
- [ ] 关键步骤异常告警

预期学习收获：

- 线上 Agent 的核心不是“偶尔很聪明”
- 核心是“长期稳定、可解释、可恢复”
